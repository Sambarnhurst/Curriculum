Lesson 23 — Small Print <small>

← Back to Curriculum ← Previous Lesson Next Lesson →
Estimated time: 25–35 minutes · Learn how to use <small> for side notes, disclaimers, and legal fine print

Learning Goals

  • Understand the semantic meaning of the <small> element.
  • Learn historical context of <small> from early HTML to HTML5.
  • Apply <small> in real-world use cases (disclaimers, footnotes, legal text).
  • Avoid accessibility pitfalls when shrinking text too far.

Part 1 — What is <small>?

The <small> element is used for side comments, disclaimers, or fine print. It signals text of lesser importance without removing it entirely. In HTML5, its semantic meaning was clarified: it’s not “make text small” — it’s “present text in a secondary or fine-print role.”

Part 2 — Example

<p>Price: $20 <small>(plus tax)</small></p>
<p><small>* Limited time offer. Terms apply.</small></p>

Result: The notes in <small> are visually smaller and clearly secondary to the main text.

Part 3 — Best Practices

  • Use for legal disclaimers, copyright notices, or notes.
  • Pair with <footer> to mark metadata about a document.
  • Never use it to shrink important content — readers with low vision may miss it.

Part 4 — Accessibility Notes

Because <small> reduces font size, ensure:

  • Contrast remains high enough for readability.
  • Critical information is never hidden in fine print.
  • Screen readers still read <small> text normally — so it’s semantic, not hidden.

Part 5 — Historical Notes

In early HTML, <small> was purely visual. In HTML5, the specification redefined it as “side comments and small print.” This change reinforced semantic use, not just styling.

Part 6 — Common Mistakes

  • ❌ Overusing <small> everywhere for styling.
  • ❌ Shrinking text too much, making it unreadable.
  • ❌ Using <small> for headings or emphasis (use <em> / <strong> instead).

Quick Quiz

  1. What is the semantic purpose of <small>?
  2. Why is accessibility a concern with <small>?
  3. What’s the difference between <small> and CSS font-size?
Sample Answers
  • To mark text as side comments, disclaimers, or fine print.
  • Small text can be hard to read if overused or styled poorly.
  • <small> has semantic meaning; CSS font-size only changes appearance.

Mini Project — Product Page with Fine Print

Build a product listing that includes:

  • Product title with price
  • A main description
  • A disclaimer in <small> (e.g., “*batteries not included*”)
  • Copyright note in a <footer> using <small>
