Lesson 1 — What is HTML? Foundations

← Back to Curriculum Next Lesson →
Estimated time: 30–45 minutes · You’ll build a simple “Hello, HTML” page

Learning Goals

  • Understand what HTML is and the role it plays on the web.
  • Recognize basic elements: <!DOCTYPE>, <html>, <head>, <body>, <title>, headings, paragraphs.
  • Build and view your first web page locally.

Part 1 — The Concept

HTML (HyperText Markup Language) provides the structure and meaning of a web page. You use elements (tags) to describe content like headings, paragraphs, links, images, and lists.

Key idea: HTML is the structure. CSS handles the presentation, and JavaScript handles the behavior.

Part 2 — Minimal Document

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>My First Page</title>
  </head>
  <body>
    <h1>Hello, HTML!</h1>
    <p>This is my first web page.</p>
  </body>
</html>

Part 3 — Common Elements

  • Headings: <h1> … <h6>
  • Paragraphs: <p>
  • Line break: <br> (use sparingly)
  • Horizontal rule: <hr>
  • Comments: <!-- note -->
  • Text emphasis: <em>, <strong>

Quick Quiz

  1. What is HTML’s main role?
  2. Which layer handles visual design?
  3. What’s the difference between a tag and an element?
Show sample answers
  • HTML’s role: structure & meaning of content (not visual design or behavior).
  • Design: CSS.
  • Tag vs element: a tag is the markup keyword in angle brackets; an element is the tag plus its content (and attributes).

Glossary

  • Tag — The markup keyword like <p>.
  • Element — The tag plus its content (and attributes).
  • Attribute — Extra info on an element: alt, href, lang.

Common Mistakes (and Fixes)

  • Missing <!DOCTYPE html> → page may render in quirks mode.
  • Forgetting lang="en" on <html> → hurts accessibility.
  • Using <br> for spacing → use paragraphs and CSS margins instead.
