Lesson 1 — What is HTML? Concept & Role

Estimated time: 45–60 minutes · You’ll make your first page

Learning Goals

  • Explain what HTML is and what problems it solves.
  • Describe how HTML relates to CSS and JavaScript.
  • Recognize tags, elements, and attributes.
  • Build and open a first HTML page on your computer.
  • Avoid the most common beginner pitfalls.

Part 1 — The Big Idea

HTML (HyperText Markup Language) is the language we use to describe the structure and meaning of content on the web. Think of it as the “skeleton” of every page.

HTML’s job

  • Organize content into headings, paragraphs, lists, images, links, tables, and forms.
  • Give content semantics—tell browsers and assistive tech what things are (e.g., a navigation, an article, a button).
Key idea: HTML is about structure & meaning, not appearance or behavior.

Not HTML’s job

  • Visual styling—that’s CSS.
  • Interactivity/logic—that’s JavaScript.

We separate concerns so each layer stays clean and powerful.

Part 2 — Where HTML Fits (HTML vs CSS vs JS)

HTML

Structure & semantics: <h1>, <p>, <img>, <nav>, <form>…

CSS

Appearance: layout, colors, spacing, typography. (We’ll add CSS later.)

JavaScript

Behavior: reacting to events, fetching data, changing the page dynamically.

Part 3 — The Anatomy of a Page

This is the minimal, valid HTML document. Read the comments inside to see what each part does.

<!DOCTYPE html> <!-- tells the browser: use modern HTML -->
<html lang="en"> <!-- root element; lang helps accessibility & SEO -->
  <head>
    <meta charset="UTF-8"> <!-- text encoding (supports emoji, accents) -->
    <meta name="viewport" content="width=device-width, initial-scale=1"> <!-- mobile-friendly -->
    <title>My First Page</title> <!-- shown in browser tab -->
  </head>
  <body>
    <h1>Hello, world!</h1>
    <p>This is my very first HTML page.</p>
  </body>
</html>
Try it: copy & save
  1. Open a text editor (Notepad, VS Code, etc.).
  2. Paste the code above into a new file.
  3. Save as my-first-page.html on your Desktop.
  4. Double-click the file to open it in your browser.

Part 4 — Tags, Elements, Attributes

  • Tags are the brackets like <p> and </p>.
  • Elements are the content + its tags (e.g., a whole paragraph).
  • Attributes add extra info inside the opening tag: <img src="cat.jpg" alt="A cat">
<h1 id="top">A Main Heading</h1>
<p class="lead">This paragraph has a class.</p>
<a href="#top" title="Back to top">Jump back up</a>

Part 5 — The DOM (Mental Model)

When a browser reads your HTML, it builds a tree of nodes called the DOM (Document Object Model). You don’t need JavaScript yet to benefit from this; it’s simply the way your page is structured in memory.

[html]
 ├─ [head]
 │   └─ [title] "My First Page"
 └─ [body]
     ├─ [h1] "Hello, world!"
     └─ [p]  "This is my very first HTML page."

Thinking in trees helps you nest elements cleanly and debug structure issues.

Part 6 — Build Your First Page (Step-by-Step)

  1. Create a folder named html-zero-to-mastery.
  2. Inside it, create a file named index.html.
  3. Paste the minimal page from Part 3.
  4. Add a heading (<h1>) with your name.
  5. Add a short paragraph (<p>) about what you want to learn.
  6. Add a link (<a>) to a favorite website.
  7. Add an image (<img>) using an online URL or a local file in the same folder.
  8. Save, then open index.html in your browser.
Starter scaffold (optional copy) 
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Your Name — First Page</title>
</head>
<body>
  <h1>Your Name</h1>
  <p>I’m learning HTML to build amazing websites.</p>

  <h2>Links I like</h2>
  <ul>
    <li><a href="https://developer.mozilla.org/">MDN Web Docs</a></li>
  </ul>

  <h2>A picture I like</h2>
  <img src="https://upload.wikimedia.org/wikipedia/commons/6/6a/JavaScript-logo.png" alt="JS Logo" width="120">
</body>
</html>

Part 7 — Open, Refresh, and Inspect

  • Open the file in your browser (double-click, or drag it into a tab).
  • Change the HTML, save (Ctrl/Cmd + S), and refresh the page.
  • Right-click the page → Inspect to open DevTools and see your HTML in the Elements panel.
Tip: If something looks wrong, the Elements panel will show what the browser actually understood.

Part 8 — Common Pitfalls

  • Forgetting closing tags (e.g., <p> without </p>).
  • Mis-nesting elements (a <p> tag around a block element like <div>).
  • Missing alt text on images.
  • Using HTML for styling (e.g., <font>)—that’s CSS’s job.
  • Missing <!DOCTYPE html> or <meta charset>.

Part 9 — Practice (Do First, Then Peek)

  1. Create a page with a main heading and two paragraphs. Make one paragraph include a link to your favorite site.
  2. Add an image with meaningful alt text. Try changing the image’s width attribute.
  3. Add a short list of 3 things you want to build with HTML.
  4. Use the lang attribute on <html> and explain why it helps.
  5. Break something on purpose (e.g., remove a closing tag) and find it using DevTools.
Hints
  • Use <h1> for the main page title, <p> for paragraphs, <a href="..."> for links.
  • <img src="..." alt="Describe the image">
  • Lists use <ul> with <li> items.
  • <html lang="en"> helps screen readers and search engines understand language.

Part 10 — Mini-Project: “Hello, I’m <Your Name>”

Build a single-page introduction with a heading, a photo (or placeholder), a short bio paragraph, and 2–3 links. Keep it pure HTML.

Project checklist
  • <!DOCTYPE html>, <html lang>, and <meta charset>.
  • A top-level <h1> with your name.
  • One photo via <img> with useful alt.
  • Two paragraphs: who you are + what you want to build.
  • Links to a resource you’ll use (docs, tutorials) and a favorite site.

<!DOCTYPE html>
<html lang="en">
<head>

Check Yourself — Quick Quiz

  1. What is the main purpose of HTML?
  2. Which layer handles visual design: HTML, CSS, or JS?
  3. What is the difference between a tag and an element?
  4. Why do we include <!DOCTYPE html>?
  5. Where does the content that actually appears on the page usually go: <head> or <body>?
  6. What is the lang attribute used for?
Suggested answers
  • HTML describes the structure and meaning (semantics) of web content.
  • CSS handles visual design; JS handles behavior.
  • Tags are the markup; an element is the tag(s) plus its content.
  • It tells the browser to use the modern standard mode (not quirks).
  • <body>.
  • It sets the page’s primary language for accessibility and search.

Glossary

  • HTML — HyperText Markup Language; the skeleton of the web.
  • Element — A piece of content with its tags, e.g., <p>hello</p>.
  • Attribute — Extra info on an opening tag, e.g., alt on <img>.
  • DOM — The in-memory tree representation of your HTML document.
  • Semantics — Meaningful tags that describe content purpose (e.g., <nav>).
