Part 1: What is Programming?

Programming is the art of giving instructions to a computer. These instructions are written in a programming language, which the computer can interpret and execute.

An algorithm is a step-by-step plan for solving a problem — much like a recipe in cooking.

  • Algorithms must be clear and precise.
  • Programs are algorithms written in code.

Part 2: Your First Program

The classic first program is Hello, World!. It simply outputs text to the screen.

print("Hello, World!")

Try changing the text inside the quotes to print your name.

Part 3: Variables and Data Types

Variables store values. Examples in JavaScript:

let name = "Alice";
let age = 25;
let isStudent = true;

Types include: Strings, Numbers, Booleans, Arrays, and Objects.

Part 4: Operators and Expressions

Operators let you perform calculations and logic:

let x = 10, y = 3;
console.log(x + y); // 13
console.log(x % y); // 1
console.log(x > y); // true

Remember: = assigns, while == compares.

Part 5: Input and Output

Programs interact using input and output.

let name = prompt("What is your name?");
console.log("Hello, " + name);

(Run this in a browser console to see prompts.)

Part 6: Conditionals

Conditionals let programs make decisions:

let age = 20;
if (age >= 18) {
  console.log("You can vote!");
} else {
  console.log("Too young to vote.");
}

Part 7: Loops

Loops repeat actions until a condition is met.

// For loop
for (let i = 1; i <= 5; i++) {
  console.log(i);
}

// While loop
let count = 5;
while (count > 0) {
  console.log(count);
  count--;
}

Part 8: Arrays and Lists

Arrays hold multiple values:

let fruits = ["Apple", "Banana", "Cherry"];
console.log(fruits[0]); // Apple
fruits[1] = "Blueberry";
console.log(fruits);

Part 9: Functions

Functions organize reusable code:

function add(a, b) {
  return a + b;
}
console.log(add(3, 4)); // 7

Part 10: Debugging

Common bug types:

  • Syntax errors: typos or missing symbols
  • Runtime errors: invalid operations (like divide by zero)
  • Logic errors: code runs but gives wrong results

Debugging tips: read error messages, print variables, test small pieces of code, and work step by step.
