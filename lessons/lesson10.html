Lesson 10 — The <title> Element

Learning Goals

  • Understand the purpose of the <title> element in HTML.
  • Learn where titles appear: browser tabs, bookmarks, search results.
  • Explore accessibility and SEO importance of titles.
  • Apply best practices for writing effective titles.

Concept

The <title> element is placed inside the <head> and sets the page’s official title. Unlike headings inside the <body>, the title does not show directly on the page — but it is crucial for how browsers, users, and search engines understand the page.

It serves four roles at once:

  • Browser Tabs: Appears in the tab/window title bar.
  • Bookmarks: Used when a page is saved/favorited.
  • Search Engines: Appears as the main clickable link in search results (SEO).
  • Accessibility: Screen readers announce the title to describe what the page is about.

Example

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>Portfolio of Jane Doe</title>
  </head>
  <body>
    <h1>Welcome to my portfolio</h1>
    <p>I build websites and design apps.</p>
  </body>
</html>

Explanation: The page title shown in the browser tab is “Portfolio of Jane Doe,” while the visible content uses <h1> and <p>.

Good vs Bad Titles

✅ Good: Short, descriptive, unique.

  • “HTML Basics — Lesson 1”
  • “John Smith | Web Developer Portfolio”

❌ Bad: Too vague or repetitive.

  • “Home” (too generic, every site has one)
  • “Untitled Document” (forgot to set title)
  • “My Website My Website My Website” (keyword stuffing)

Common Mistakes

  • Leaving the title blank → browser shows “Untitled.”
  • Using the same title for all pages → confuses users and hurts SEO.
  • Making it too long → search engines cut it off after ~60 characters.
  • Placing <title> in the <body> instead of the <head>.

Quick Quiz

  1. Where should the <title> element be placed?
  2. How does the title affect search engine results?
  3. Why is it important for accessibility?
  4. True/False: You can use multiple <title> elements on one page.
Answers
  • Inside the <head> section.
  • It becomes the clickable link in search results and signals relevance.
  • Screen readers announce it to help users identify the page.
  • False — only one <title> per page.

Did You Know?

The very first webpage in 1991 had the title “The World Wide Web project.” This simple <title> still works today and is considered one of the most historic HTML tags ever used.

FAQ

Q: Can I include branding in every title?
A: Yes, but add unique page info before or after (e.g., “Blog — MyBrand”).

Q: Should I use ALL CAPS in titles?
A: Avoid it — it can look like shouting and reduce readability.

Q: Do titles impact Google ranking?
A: Yes, they are one of the strongest on-page SEO signals.
