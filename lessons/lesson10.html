Lesson 10 — The <title> Element

← Back to Curriculum Next Lesson →

Learning Goals

  • Understand the purpose of the <title> element.
  • See where the title appears in browsers and search engines.
  • Learn best practices for writing good titles.

Concept

The <title> element defines the title of a web page. It appears in:

  • The browser tab or window.
  • Bookmarks and history lists.
  • Search engine results.

Although it’s not visible in the page content, it is one of the most important elements for usability, SEO, and accessibility.

Example

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>My Awesome Page</title>
  </head>
  <body>
    <h1>Welcome to my site!</h1>
    <p>This page has a proper title in the browser tab.</p>
  </body>
</html>

Explanation: The <title> sets the browser tab text. Here it would show “My Awesome Page.”

Common Mistakes

  • Forgetting the <title> (leads to “Untitled” pages in the browser).
  • Using the same title on every page (bad for navigation & SEO).
  • Making titles too long or too short (best is 50–60 characters).

Quick Quiz

  1. Where does the <title> element appear?
  2. Is <title> placed in the <head> or <body>?
  3. Why is the <title> important for search engines?
Answers
  • In the browser tab, bookmarks, and search engine results.
  • Inside the <head>.
  • It tells search engines what the page is about.

Mini Project

Create a new HTML page and:

  • Set the <title> to your name + “Portfolio.”
  • Add a heading and paragraph in the <body>.
  • Open it in your browser and confirm the tab shows your custom title.

Did You Know?

The <title> tag is one of the earliest parts of HTML (present since 1991). Even though it looks small, it plays a huge role in web search ranking and user trust.

FAQ

Q: Can I put emojis in a <title>?
A: Yes, modern browsers support Unicode, but use them sparingly.

Q: What happens if I skip the <title>?
A: The browser may show the file name instead, which looks unprofessional.
