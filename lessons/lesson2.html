Lesson 2 — HTML vs CSS vs JavaScript Layer Roles & Separation

Estimated time: 60–75 minutes · Build a complete layered page

Learning Goals

  • Explain roles of HTML (structure), CSS (presentation), JavaScript (behavior).
  • Apply separation of concerns using distinct files and clean hooks.
  • Understand the DOM, CSSOM, and Render Tree flow.
  • Implement a theme toggle that saves preference.
  • Avoid inline styles and inline event handlers.

Part 1 — The Big Idea

Build the web in layers so each concern stays focused and maintainable:

What each layer does

  • HTML: structure & semantics — headings, paragraphs, forms, landmarks.
  • CSS: visuals — layout, spacing, color, typography, responsive rules.
  • JavaScript: behavior — events, dynamic updates, state, persistence.

What not to mix

  • Avoid styling with HTML attributes or deprecated tags (e.g., <font>).
  • Avoid onclick="..." et al. — add listeners in JS instead.

Separation improves readability, testing, performance, and teamwork.

Part 2 — Where Each Layer Fits

HTML

Semantic tags like <header>, <main>, <article>, <nav>, <button>.

CSS

Color palette, spacing scale, grids/flex, typography, motion.

JS

Events, toggles, persistence (localStorage), fetching data.

Part 3 — Minimal Triad Example

Each layer in the right place:

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1">
    <title>Triad Example</title>
    <link rel="stylesheet" href="styles.css">
  </head>
  <body>
    <header>
      <h1>Hello, Triad!</h1>
      <button id="themeBtn">Toggle Theme</button>
    </header>
    <main>
      <article>
        <h2>Why separation of concerns?</h2>
        <p>Structure, style, and behavior live in their own layers.</p>
      </article>
    </main>
    <script src="app.js" defer></script>
  </body>
</html>

Part 4 — Terms & Mental Model

HTML  →  Parser  →  DOM Tree
CSS   →  Parser  →  CSSOM Tree
DOM + CSSOM → Render Tree → Layout → Paint → Composite
JS can read/modify DOM & CSSOM and trigger reflow/paint when needed.

Use DevTools → Elements & Computed to see final styles and the live DOM.

Part 5 — Build Your Layered Page (Step-by-Step)

  1. Create folder lesson-02.
  2. Create index.html, styles.css, app.js.
  3. Paste the HTML from Part 3 into index.html.
  4. Add base styles & theme modifiers in styles.css.
  5. Implement theme toggle in app.js and persist to localStorage.
  6. Open in browser and test.
Starter CSS 
:root { color-scheme: light dark; }
body { font-family: system-ui, sans-serif; margin: 0; line-height: 1.6; padding: 1.25rem; }
header { display: flex; gap: .75rem; align-items: center; justify-content: space-between; }
button { padding: .5rem .75rem; border: 1px solid #ccc; border-radius: .5rem; background: #f4f4f4; cursor: pointer; }
main { max-width: 60ch; }
.dark body { background: #121212; color: #eaeaea; }
.light body { background: #ffffff; color: #222; }
Starter JS 
const btn = document.getElementById('themeBtn');
const saved = localStorage.getItem('theme');
if (saved) document.documentElement.className = saved;
btn.addEventListener('click', () => {
  const root = document.documentElement;
  const next = root.classList.contains('dark') ? 'light' : 'dark';
  root.className = next;
  localStorage.setItem('theme', next);
});

Part 6 — Open, Refresh, and Inspect

  • Open index.html in your browser.
  • Edit files, save (Ctrl/Cmd + S), refresh to see changes.
  • Use DevTools → Console to try document.querySelector('h1').textContent.
Tip: If the toggle doesn’t work, check that the script tag has defer and open Console for errors.

Part 7 — Common Anti-Patterns

❌ Inline styles

<p style="color:red;font-size:24px">Hi</p>

Why it hurts: hard to reuse/override; mixes concerns.

✅ Use classes + CSS

<p class="warning">Hi</p>
/* CSS */ .warning { color:red; font-size:24px; }

❌ Inline event handlers

<button onclick="doThing()">Click</button>

✅ Add listeners in JS

<button id="doBtn">Click</button>
<script>
  document.getElementById('doBtn').addEventListener('click', () => alert('Done!'));
</script>

Part 8 — Practice (Do First, Then Peek)

  1. Add a nav with two links; ensure it’s readable with JS disabled.
  2. Style the page title using CSS only (no inline styles).
  3. Add a button that reveals a hidden paragraph via JS (hidden attribute).
  4. Temporarily remove the stylesheet link; use DevTools to diagnose the impact.
Hints
  • Navigation is structural (HTML).
  • Prefer classes for styling hooks.
  • elem.hidden = !elem.hidden; toggles visibility.
One possible solution 
<nav><a href="#">Home</a> · <a href="#">Docs</a></nav>
<h1 class="title">Layered Page</h1>
<p id="extra" hidden>Surprise content!</p>
<button id="revealBtn">Reveal</button>
/* CSS */
.title { color:#00ffff; font-weight:800; }
#extra[hidden]{ display:none; }
// JS
const extra = document.getElementById('extra');
document.getElementById('revealBtn').addEventListener('click', () => {
  extra.hidden = !extra.hidden;
});

Check Yourself — Quick Quiz

  1. Which layer defines a page’s main heading and navigation?
  2. Which layer changes colors and spacing?
  3. Which layer responds when a button is clicked?
  4. Why are inline styles a code smell?
  5. Difference between the DOM and the CSSOM?
Suggested answers
  • HTML.
  • CSS.
  • JavaScript.
  • They mix concerns, are hard to override/reuse, and hurt maintainability.
  • DOM = HTML structure; CSSOM = parsed CSS rules; both form the render tree.

Glossary

  • Separation of Concerns — Structure, style, behavior in distinct layers.
  • DOM — Tree representation of HTML nodes.
  • CSSOM — Tree of CSS rules and cascade.
  • Render Tree — DOM + CSSOM merged for layout and paint.
