Lesson 2 — HTML vs CSS vs JavaScript

Estimated time: 60–75 minutes · You’ll build a layered page and refactor anti-patterns

Learning Goals

  • Explain the role of each layer: HTML (structure), CSS (presentation), JS (behavior).
  • Apply separation of concerns to a small page.
  • Recognize and refactor common anti-patterns (inline styles, inline event handlers).
  • Understand how browsers build the DOM, CSSOM, and Render Tree.
  • Practice a tiny feature: theme toggle implemented cleanly.

Part 1 — The Three-Layer Cake

HTML — Structure

Headings, paragraphs, lists, images, forms, semantic landmarks.

CSS — Presentation

Layout, spacing, colors, fonts, responsive rules, animations.

JS — Behavior

Event handling, dynamic updates, data fetching, state.

Mental model: HTML should be readable and useful even without CSS/JS. CSS makes it beautiful. JS makes it interactive.

Part 2 — A Minimal Triad Example

Here’s a tiny site showing the three layers working together.

index.html (HTML) 
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <meta name="viewport" content="width=device-width, initial-scale=1">
  <title>Triad Example</title>
  <link rel="stylesheet" href="styles.css">
</head>
<body>
  <header>
    <h1>Hello, Triad!</h1>
    <button id="themeBtn">Toggle Theme</button>
  </header>

  <main>
    <article>
      <h2>Why separation of concerns?</h2>
      <p>Keeping structure, style, and behavior separate makes code easier to read, test, and maintain.</p>
    </article>
  </main>

  <script src="app.js" defer></script>
</body>
</html>
styles.css (CSS) 
/* Base style */
:root { color-scheme: light dark; }
body { font-family: system-ui, sans-serif; margin: 0; line-height: 1.6; padding: 1.25rem; }
header { display: flex; gap: .75rem; align-items: center; justify-content: space-between; }
button { padding: .5rem .75rem; border: 1px solid #ccc; border-radius: .5rem; background: #f4f4f4; cursor: pointer; }
main { max-width: 60ch; }

/* Theme modifier */
.dark body { background: #121212; color: #eaeaea; }
.light body { background: #ffffff; color: #222; }
app.js (JS) 
const btn = document.getElementById('themeBtn');
// Store preference
const saved = localStorage.getItem('theme');
if (saved) document.documentElement.className = saved;

btn.addEventListener('click', () => {
  const root = document.documentElement;
  const next = root.classList.contains('dark') ? 'light' : 'dark';
  root.className = next;
  localStorage.setItem('theme', next);
});

Part 3 — Render Pipeline (Mental Diagram)

HTML  →  Parser  →  DOM Tree
CSS   →  Parser  →  CSSOM Tree
DOM + CSSOM → Render Tree → Layout → Paint → Composite
JS can read/modify DOM & CSSOM and trigger reflow/paint when needed.

You’ll use DevTools to view computed styles and the DOM to debug mismatches.

Part 4 — Anti-Patterns to Avoid

❌ Inline styles

<p style="color:red;font-size:24px">Hi</p>

Problems: hard to reuse, hard to override, mixes concerns.

✅ Move to CSS

<p class="warning">Hi</p>
.warning { color: red; font-size: 24px; }

❌ Inline event handlers

<button onclick="doThing()">Click</button>

✅ Add listeners in JS

<button id="doBtn">Click</button>
<script>
  document.getElementById('doBtn').addEventListener('click', doThing);
  function doThing(){ alert('Done!'); }
</script>

Part 5 — Hands-On: Refactor a Messy Page

Start messy, then separate the concerns.

1) Messy single-file HTML 
<!DOCTYPE html>
<html lang="en">
<head><meta charset="UTF-8"><title>Messy</title></head>
<body style="background:black;color:white">
  <h1 style="color:#0ff">Profile</h1>
  <button onclick="alert('Hi')" style="padding:8px;border:1px solid #0ff">Say hi</button>
</body>
</html>
2) Clean HTML + CSS + JS 
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8">
  <title>Clean Profile</title>
  <link rel="stylesheet" href="clean.css">
</head>
<body>
  <h1 class="title">Profile</h1>
  <button id="hiBtn" class="btn">Say hi</button>
  <script src="clean.js" defer></script>
</body>
</html>
/* clean.css */
body{ background:#121212; color:#eaeaea; font-family:system-ui,sans-serif; }
.title{ color:#00ffff; }
.btn{ padding:8px; border:1px solid #00ffff; background:#1c1c1c; color:#eaeaea; border-radius:8px; }
// clean.js
document.getElementById('hiBtn').addEventListener('click', () => alert('Hi'));

Part 6 — DevTools Micro-Lab

  1. Elements: Inspect a heading and confirm which rules style it.
  2. Computed: Check the final color and font-size of a paragraph.
  3. Console: Run document.querySelector('h1').textContent and change it.
  4. Sources: Add a breakpoint in JS and click a button.

Part 7 — Mini-Project: Profile Card (Layered)

Create a simple profile card that looks good with CSS and has a small JS interaction (e.g., toggling extra details).

Starter HTML 
<section class="card profile">
  <h2>Your Name</h2>
  <p>Short bio line here.</p>
  <button id="moreBtn">Show more</button>
  <div id="more" hidden>
    <p>More details about you...</p>
  </div>
</section>
CSS & JS sketch 
.profile { max-width: 420px; margin: 1rem auto; background:#1c1c1c; padding:1rem; border-radius:12px; border:1px solid #2a2a2a; }
#more[hidden]{ display:none; }
const more = document.getElementById('more');
document.getElementById('moreBtn').addEventListener('click', () => {
  more.hidden = !more.hidden;
});

Check Yourself — Quick Quiz

  1. Which layer should define a page’s main heading and nav structure?
  2. Which layer should change colors and spacing?
  3. Which layer should respond when a button is clicked?
  4. Why are inline styles considered a smell?
  5. What’s the difference between the DOM and the CSSOM?
Suggested answers
  • HTML.
  • CSS.
  • JavaScript.
  • They mix concerns, are hard to override and reuse, and hurt maintainability.
  • DOM represents the HTML structure; CSSOM represents parsed CSS rules. Both combine to build the render tree.

Glossary

  • Separation of Concerns — Keeping structure, style, and behavior in their own layers.
  • DOM — Tree representation of HTML nodes.
  • CSSOM — Tree of CSS rules and their specificity cascade.
  • Render Tree — Combination used to layout and paint pixels.
