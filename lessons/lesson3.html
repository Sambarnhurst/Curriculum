Lesson 3 ‚Äî Browsers & How They Read HTML

Learning Goals

  • Understand how browsers load and parse an HTML file.
  • Learn about the rendering process (DOM, CSSOM, Render Tree).
  • Recognize why clean, valid HTML is important.

1 ‚Äî The Browser‚Äôs Job

When you type a URL or open an HTML file, the browser:

  1. Fetches the HTML file.
  2. Parses it into a tree-like structure (the DOM).
  3. Loads resources referenced in HTML (CSS, JS, images).
  4. Builds the page combining DOM (structure), CSSOM (styles), and JS (behavior).
  5. Paints the final pixels on your screen.

2 ‚Äî Example: Simple HTML ‚Üí Render

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="UTF-8">
    <title>Browser Demo</title>
    <style>
      body { background:#121212; color:#eee; }
      h1 { color: cyan; }
    </style>
  </head>
  <body>
    <h1>Browser Rendering</h1>
    <p>The browser parses HTML into a DOM, applies CSS, and executes JS.</p>
    <script>
      console.log("Page loaded!");
    </script>
  </body>
</html>

The browser turns this code into a structured page with styled text, then runs the script.

3 ‚Äî Why This Matters

  • Broken HTML ‚Üí unpredictable rendering.
  • Invalid nesting ‚Üí browser may auto-correct incorrectly.
  • Performance ‚Üí fewer blocking resources = faster rendering.
Tip: Use browser DevTools (F12) to see the DOM the browser created.

Practice Task

  1. Create a simple HTML file with a heading, paragraph, and image.
  2. Open the DevTools ‚ÄúElements‚Äù tab to inspect the DOM structure.
  3. Try breaking a tag (like leaving off </p>) and see how the browser fixes it.
  4. Check the ‚ÄúConsole‚Äù tab to see any errors or messages.
